# 熙康云医院飞书客户案例分析报告

**公司名称**: 熙康云医院（股份代号：9686.HK）

**行业**: 医疗健康/互联网医疗

**规模**: 501-2000人

**主营业务**:
- 中国城市云医院平台模式的创造者和推动者

- 围绕居民健康需求，推动"护理+医疗+健康"服务场景和内容创新

- 中国首个城市云医院平台

## 2. 飞书应用场景

### 2.1 体检用工数字化管理
- 利用飞书多维表格和审批流功能
- 建立涵盖人员管理、流程控制、自动化核算到成本监控的全流程自动化管理体系
- 解决体检业务中人员管理的复杂性和特殊需求

### 2.2 需求管理数字化
- 采用飞书多维表格功能克服原有项目管理软件在需求收集、进度同步以及版本追踪等方面的低效难题
- 自动发送需求收集表单，简化需求提交过程
- 利用飞书强大的版本管理和需求关联功能
- 结合自动化消息推送机制，确保所有相关人员实时掌握开发进度与上线时间

### 2.3 业务数据管理
- 飞书多维表格与现有业务系统的API进行实时同步，实现数据自动更新
- 提供多维度可视化的数据支持
- 构建全链路透明的数据管理体系

## 3. 新获得的能力

### 3.1 协作效率提升
- **协作效率提升80%**: 通过数字化手段将用工管理进行全面升级
- 实现透明、高效、规范的管理模式
- 多个流程间的无缝对接和信息的高效流转

### 3.2 数据查询效率提升
- **业务数据查询时间从30分钟缩短至1分钟**: 大幅提升数据驱动决策的效率
- 减少90%的人工干预需求
- 提升数据获取的速度和准确性

### 3.3 移动办公能力
- 需求方随时随地查询进展
- 克服原系统主要针对PC端设计的局限性
- 满足外出频繁的需求方便捷沟通的需求

### 3.4 自动化管理能力
- 需求收集自动化
- 数据自动更新
- 自动化消息推送机制
- 成本监控自动化

### 3.5 决策支持能力
- 多维度可视化的数据支持
- 管理层即时洞察关键指标
- 大幅提升决策效率

## 4. 未满足的需求

### 4.1 原有系统局限性
- **移动端支持不足**: 原有项目管理软件主要针对PC端设计，难以满足外出频繁的需求方便捷沟通
- **信息交流不便**: 传统系统导致信息交流不便及操作流程繁琐
- **版本追踪困难**: 原有系统在需求收集、进度同步以及版本追踪等方面效率低下

### 4.2 数据处理瓶颈
- **传统数据查询效率低**: 单次查询需要30分钟，严重影响决策效率
- **人工干预多**: 原有数据处理模式需要大量人工干预
- **数据可视化不足**: 缺乏多维度可视化的数据支持

### 4.3 管理模式局限
- **传统用工管理低效**: 以往低效的传统管理模式无法满足现代医疗服务的复杂需求
- **流程孤立**: 多个流程间缺乏无缝对接和信息高效流转
- **成本监控困难**: 缺乏实时、精准的成本监控能力
