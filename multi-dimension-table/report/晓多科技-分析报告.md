# 晓多科技飞书客户案例分析报告

## 1. 客户基本信息

- **公司名称**：晓多科技有限公司
- **所属行业**：企业服务/SaaS行业，认知智能解决方案
- **公司规模**：350余人的"前线部队"（销售、客户成功、客服）
- **成立时间**：2014年
- **主营业务**：以智能对话为核心的认知智能SaaS产品和行业数智化解决方案服务商，核心产品为智能客服机器人
- **业务覆盖**：全国50多个城市，累计服务38000+客户
- **行业拓展**：从纵向深耕电商行业，到横向拓展到电信、政务、高校、教育等行业

## 2. 飞书给客户带来的新能力和价值

### 2.1 具体使用的飞书功能模块

1. **飞书服务台**
   - 搭建了"技术支持服务台"
   - 基于精准的语义识别自动应答常规问题
   - 沉淀销售、客户成功、客服等前线员工反复询问的问题为知识库

2. **飞书开放平台**
   - 接入14个内部和外部系统的应用API
   - 实现系统间的无缝集成
   - 自动以飞书账号登录其他系统

3. **飞书IM生态**
   - 接收各系统关键信息和通知的推送
   - 技术部门内部紧急群组接收服务器告警信息
   - 新人入职自动接收业务系统账号开通推送

4. **飞书文档**
   - 用于知识库建设和信息沉淀

5. **飞书招聘**
   - 支持招聘流程管理

6. **飞书绩效**
   - 支持绩效管理流程

### 2.2 解决的业务痛点

1. **前线员工支持效率问题**
   - **痛点**：350余人的前线部队（销售、客户成功、客服）面对各行业客户的不同疑问，经常需要向研发、产品经理等后端同事寻求支援，存在大量重复性问题
   - **解决方案**：通过飞书服务台搭建技术支持服务台，90%的问题自动应答，减少人工干预

2. **多系统登录繁琐问题**
   - **痛点**：采买和自建了14个系统，员工需要记住每个账号密码，登录操作繁琐
   - **解决方案**：通过飞书开放平台API接入，14个系统无需输入账号密码即可在飞书内打开

3. **信息安全和风险控制问题**
   - **痛点**：员工离职后仍可能登录公司系统，造成信息泄露风险
   - **解决方案**：关闭飞书账号即可自动关闭其他用飞书登录的业务系统账号

### 2.3 带来的业务价值和效率提升

1. **技术服务效率提升**
   - 350余人的前线团队，90%的问题在技术支持服务台上自助解决
   - 仅需3位员工解决剩余10%的非常规性问题
   - 技术支持服务台一年提效55个"工日"
   - 一年为晓多科技节省8万元成本

2. **系统整合和统一登录**
   - 原来14个系统账号登录简化为只需登录飞书1个账号
   - 实现All in one，用飞书就能"一键登录"
   - 飞书成为晓多内部信息流动的"总阀门"

3. **跨部门服务台扩展**
   - 除技术支持外，还搭建了行政、HR、财务、IT等部门的服务台
   - 让更多问题能够得到自助解答

### 2.4 具体数据支撑

- **90%**：技术服务台能解决的重复性问题比例
- **8万元**：技术服务台一年为晓多科技节省的成本
- **55个"工日"**：技术服务台一年提效的工作量
- **14→1**：原来14个系统账号简化为只需1个飞书账号
- **350+人**：前线团队规模
- **3人**：解决非常规性问题的员工数量
- **50+城市**：业务覆盖范围
