# 云智慧-分析报告

## 第126家客户分析报告

## 1. 客户基本信息

**公司名称**: 云智慧（北京）科技有限公司
**所属行业**: 企业服务/智能运维
**公司规模**: 未明确提及，但从业务覆盖范围推测为大型企业
**主营业务**: 全栈智能业务运维解决方案服务商，提供从ITOM到ITSM的智能运维产品系列

**公司背景**: 云智慧成立于2009年，是全栈智能业务运维解决方案服务商。经过多年自主研发，已经形成了从ITOM到ITSM的智能运维产品系列，为金融、政府、运营商、能源等数十个行业客户提供了数字化运维体系建设，以及生命周期运维管理解决方案。据IDC数据，2022年，其ITSM软件在中国IT统一运维软件领域市场份额排名第一。目前，云智慧也推出了高效、开箱即用的IT服务管理新工具——云智慧数字化运维服务管理平台ITSM SaaS版产品。该产品很快也会上架飞书应用商城。

## 2. 飞书给客户的新能力/赋能

### 2.1 系统集成与数据流动能力

**新能力**: 通过飞书集成平台打通CRM客户关系管理系统和PMO项目管理系统
- **具体应用**: 实现系统之间的数据透传，将飞书多维表格搭建成轻量敏捷的客户管理系统
- **价值提升**: 打破了原有系统间的数据孤岛，实现了信息的无缝流转

### 2.2 销售流程移动化管理能力

**新能力**: 飞书移动端审批+CRM系统集成
- **具体应用**: 销售人员在飞书移动端即可随时随地发起审批，将信息和数据及时快速录入CRM系统
- **价值提升**: 一线销售人员得以快速地推进、跟进和处理商机线索，大大提高了销售流程的效率和精准度

### 2.3 项目进度透明化管理能力

**新能力**: 飞书集成平台+多维表格的项目管理解决方案
- **具体应用**: 项目任务进度被自动化同步到多维表格中，客户在一张表上查看所有项目进度
- **价值提升**: 客户需求对应的开发任务进度和工时信息实现自动化同步，替代了原有的Excel、截图手动导出方式

### 2.4 销售数据可视化与决策支持能力

**新能力**: 飞书多维表格仪表盘+定制工作台的数据看板
- **具体应用**: 将CRM系统中的销售数据同步至多维表格，使用仪表盘能力对销售数据进行可视化处理
- **价值提升**: 打造出销售数据管理舱，让数据主动流向对的人，管理层可以及时获取和分析重要信息

### 2.5 区域化业绩管理能力

**新能力**: 分类销售大区管理+定制化工作台
- **具体应用**: 分类为数十个销售大区分别搭建本区域"销售监测工作台"，无需代码10分钟即可打造业绩数据通晒工作台
- **价值提升**: 合同额完成情况、年度目标达成进度、各部门排名与大客户分布等核心数据一目了然

### 2.6 员工激励与表彰能力

**新能力**: 业绩通晒与销售冠军展示
- **具体应用**: 业绩通晒、销售冠军名列前茅的信息展示
- **价值提升**: 给员工带来了表彰激励，激发销售团队积极性

## 3. 飞书没有满足客户需求的部分

### 3.1 原有CRM系统移动端体验不足

**未满足需求**: 原有CRM系统在移动端的体验不佳
- **具体表现**: 销售人员每次更新客户信息、跟进记录时，都需要使用电脑进入CRM系统中填报，麻烦费时
- **解决方案**: 需要通过飞书移动端审批来补充，而非直接优化原CRM系统

### 3.2 原有项目管理系统的客户可见性不足

**未满足需求**: 现有PMO系统进行项目管理时，客户无法直接查看项目进度
- **具体表现**: 需要用Excel、截图的方式手动导出项目信息并传递给客户
- **解决方案**: 通过飞书集成平台和多维表格来弥补，但并未改造原系统

### 3.3 原有CRM系统数据分析能力有限

**未满足需求**: 销售数据封闭在CRM系统中，无法根据自定义规则进一步计算业绩结果
- **具体表现**: 管理层不能及时方便地获取到业绩信息，也很难根据业绩进展及时调整销售策略
- **解决方案**: 需要将数据导出到飞书多维表格进行处理和分析

### 3.4 系统整合深度不足

**未满足需求**: 飞书主要是作为原有系统的"中间层"或"展示层"，而非深度整合
- **具体表现**: 数据需要在多个系统间流转，增加了系统复杂性
- **潜在风险**: 可能存在数据同步延迟或不一致的问题

## 4. 深入的分析洞察

### 4.1 企业服务行业的数字化转型特征

云智慧案例体现了企业服务行业数字化转型的典型特征：
- **轻量化改造优先**: 选择通过飞书等轻量级工具进行功能补充，而非全面替换现有系统
- **移动办公需求迫切**: 一线销售人员的移动办公需求是驱动数字化改造的重要动力
- **数据可视化决策**: 从经验驱动转向数据驱动的决策模式转变

### 4.2 飞书在企业服务场景中的定位

飞书在云智慧案例中扮演了"数字化连接器"的角色：
- **系统集成枢纽**: 通过集成平台连接多个专业系统，实现数据互通
- **移动办公入口**: 提供移动端的便捷操作界面，补充原有系统的不足
- **数据可视化平台**: 通过多维表格和仪表盘提供数据分析和展示能力

### 4.3 敏捷CRM建设的创新模式

云智慧构建了一个"敏捷CRM系统"的创新模式：
- **低代码开发**: 利用飞书多维表格等工具快速构建应用，无需传统编程
- **渐进式改造**: 不替换原有系统，而是在其基础上增加新功能
- **用户体验优先**: 以销售人员的需求和使用体验为中心设计流程

### 4.4 数据驱动销售的实践路径

云智慧展示了数据驱动销售的具体实践路径：
- **数据采集**: 从CRM系统中自动同步销售数据
- **数据加工**: 在多维表格中进行自定义计算和分析
- **数据展示**: 通过仪表盘和工作台进行可视化展示
- **数据应用**: 支持管理决策和员工激励

### 4.5 企业服务公司的自用数字化转型意义

作为一家企业服务公司，云智慧自身的数字化转型具有特殊意义：
- **产品验证**: 通过自身使用验证产品理念和技术路线
- **案例示范**: 为客户提供可参考的数字化转型案例
- **生态融合**: 与飞书等平台生态深度融合，增强产品竞争力

### 4.6 飞书生态系统的价值体现

云智慧案例展示了飞书生态系统的核心价值：
- **平台化能力**: 通过集成平台、多维表格等基础能力支持多样化应用场景
- **低代码开发**: 使企业能够快速构建符合自身需求的应用系统
