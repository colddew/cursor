# 熹乐科技：非常时期的飞书应用分析报告

## 1. 公司基本信息

**公司名称**：北京熹乐科技有限公司
**行业**：工业互联网/轻工行业
**规模**：未明确说明，但从描述可推断为中小型科技公司
**主营业务**：
- 聚焦工业互联网领域，专注于轻工行业
- 推动AI和云计算技术与工业场景的融合
- 助力传统产业龙头企业进行数字化升级
- 核心产品：熹乐产业舆图，记录中国传统行业数字化进程
**团队分布**：北京和厦门两地办公
**成立背景**：2018年末进入产业互联网和工业互联网赛道
**历史成就**：曾带领团队打造多款影响过亿用户的科技产品

## 2. 飞书应用场景

### 2.1 核心理念：工具箱"轴心"战略
熹乐科技将飞书定位为企业协作的"轴心"，通过飞书连接所有协作工具，形成统一的协作生态体系。

### 2.2 具体应用场景

#### 远程办公协作
- **背景**：疫情期间长达一个多月的全员远程办公
- **解决方案**：飞书日历的团队日历功能，高效安排会议时间
- **效果**：无需挨个询问参会人员时间，大幅提升排会效率

#### 产品开发流程优化
- **挑战**：UI设计流程"小、散、杂"，耗费大量时间和精力
- **解决方案**：基于飞书Bot生态，将传统CMS类内部产品迁移到飞书
- **效果**：减少UI开发需求，降低整体架构和协作复杂度

#### 招聘流程自动化
- **背景**：业务快速发展，团队压力增加，需要加快招聘速度
- **解决方案**：打造虚拟员工（飞书Bot）代替招聘中的重复性劳动
- **效果**：提升招聘效率，实现"逆势招聘"

### 2.3 技术架构应用

#### 消息中心整合
- 将各类工具产生的消息统一汇集到飞书平台
- 实现内容/信息的分类分流和自动化处理
- 解决信息断层和信息孤岛问题

#### 文档协作系统
- 支持多人同时协作编写
- 实现内容与团队成员的直接链接
- 替代传统的Excel、PPT、Word和PDF文档流程

#### Bot生态系统
- 自主开发多个功能型Bot解决特定问题
- 应用Serverless平台与飞书开放平台深度集成
- 通过"雇佣"虚拟员工提升效率

## 3. 新获得的能力

### 3.1 协作效率提升
- **团队协作升级**：从1.0工具箱升级到2.0版本，实现"简单、可连接、自动化"的核心理念
- **成本显著降低**：整体协作成本节省70%
- **结构优化**：实现最简洁的工具箱结构

### 3.2 远程协作能力
- **异步协作文化**：减少团队整体上下文切换(Context Switching)，将单点效率发挥到最大
- **视域焦虑管理**：通过统一信息汇集和流动，降低远程办公带来的管理焦虑
- **多地协同**：成功支持北京和厦门两地的远程协作

### 3.3 自动化与智能化
- **Bot自动化能力**：自主开发多个功能性Bot，自动化处理重复性工作
- **信息处理自动化**：自动抓取、整理、分类多维度信息
- **系统集成能力**：通过飞书开放平台连接各个平台信息

### 3.4 产品开发能力
- **快速原型开发**：基于飞书生态快速打造产品原型
- **架构简化**：减少传统UI开发需求，专注于核心功能
- **敏捷迭代**：利用Bot生态支持快速产品迭代

### 3.5 数据处理与分析
- **多源信息整合**：将分散的工业互联网信息汇集整理
- **监控与报告**：Google Analytics Bot自动发送各种环境监控日报
- **数据驱动决策**：基于实时数据指导运营和产品迭代

## 4. 未满足的需求

### 4.1 AI能力局限
- **自然语言处理限制**：中文环境下自然语言处理、关键信息提取等AI能力还有很大提升空间
- **人工干预需求**：为保证质量，机器处理后仍需运营人员人工介入
- **自动化程度不足**：当前阶段无法实现完全自动化处理

### 4.2 技术挑战
- **Bot开发门槛**：需要技术能力才能开发适合的Bot解决方案
- **系统集成复杂性**：各类工具与飞书的深度集成仍存在技术挑战
- **质量保证机制**：需要建立完善的Bot质量评估和改进机制

### 4.3 协作文化适应
- **管理理念转变**：从传统管理向异步协作文化的转变需要时间
- **工具使用习惯**：团队成员需要适应新的协作工具和工作方式
- **信任建立**：远程协作环境下的团队信任建立需要过程

### 4.4 行业特定需求
- **工业互联网专业性**：轻工行业特定需求可能需要更多定制化解决方案
- **信息孤岛问题**：工业互联网信息分散问题仍需更多技术手段解决
- **行业知识库建设**：需要更多行业专业知识沉淀和组织
