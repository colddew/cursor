# 跬智信息飞书客户案例深度分析报告

## 公司基本信息

**公司名称：** 跬智信息（Kyligence）

**行业：** 大数据分析与指标平台服务

**成立时间：** 2016年

**核心业务：**
- 企业级OLAP（多维分析）平台产品Kyligence Enterprise
- 一站式指标平台Kyligence Zen
- 为用户提供企业级经营分析能力、决策支持系统及各种基于数据驱动的行业解决方案

**服务客户：** 中国、美国、欧洲及亚太的多个银行、证券、保险、制造、零售、医疗等行业客户

**关键人物：** 陈晓杰 - 全球云业务副总裁

## 飞书应用场景

### 1. 工时管理系统
- **工具组合：** 飞书多维表格 + 自有Kyligence Zen指标平台
- **应用部门：** 客户项目管理和产品研发部门
- **核心功能：** 通过飞书多维表格的"表单视图"功能，一线员工便捷填报、提交个人工时数据

### 2. 产品交付管理
- **工具组合：** 飞书多维表格 + Kyligence Zen + JIRA集成
- **核心功能：** 设置配置表管理产品版本、交付日期，整合研发计划数据，自动推送达成率至IM

### 3. PoC审核管理
- **工具组合：** 飞书审批流程
- **核心功能：** 多方协作审核，规范PoC流程，预防资源浪费

### 4. 知识管理
- **工具组合：** 飞书知识库 + 全局搜索功能
- **核心功能：** 统一沉淀专业知识，加速售前业务了解

## 新获得的能力

### 1. 精细化工时管理能力
- **量化管理：** 量化每一个项目的投入和产出，了解项目盈亏状况
- **过程可视化：** 实时了解产研部门时间分配（新功能研发vs产品bug修复）
- **资源优化：** 确保人力投入在最重要的事上

### 2. 产品研发过程透明化
- **配置管理：** 产品版本、交付日期信息一目了然
- **风险预警：** 通过达成率指标自动推送，及时发现延期风险
- **即时干预：** 产研管理者可及时介入降低风险

### 3. PoC流程规范化
- **多层审核：** 业务部门leader + 解决方案团队双重审核
- **方案验证：** 确保PoC方案合理性和必要性
- **资源保护：** 预防资源过度投入和浪费

### 4. 知识沉淀与共享
- **统一管理：** 将分散在员工电脑、业务部门的信息集中管理
- **快速检索：** 通过全局搜索功能快速查找信息
- **知识流转：** 售前培养周期从6-12个月缩短至3个月，知识流转速度提升50%

## 未满足的需求

### 1. 工时管理深度分析需求
- **暗示需求：** 案例中提到"了解产研部门有多少时间花在新功能研发上，有多少时间花在产品bug修复上"，暗示需要更细粒度的工时分类和分析能力
- **潜在局限：** 当前系统可能还缺乏工时数据的智能分析和预测功能

### 2. 跨系统集成深度需求
- **当前状态：** 已经实现了飞书与JIRA、自有Kyligence Zen的集成
- **潜在需求：** 大数据分析平台企业可能需要更多专业开发工具和项目管理工具的深度集成

### 3. AI智能决策支持
- **行业特性：** 作为大数据分析平台供应商，跬智信息可能期望更智能的决策支持功能
- **潜在需求：** 工时数据与业务表现的智能关联分析、资源分配的智能建议等

### 4. 全球化协作需求
- **客户覆盖：** 服务中国、美国、欧洲及亚太客户
- **潜在需求：** 多语言、多时区协作支持可能还不够完善
