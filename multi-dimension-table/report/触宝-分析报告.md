# 触宝飞书客户案例深度分析报告

触宝（CooTek）

### 所属行业

## 2. 飞书给客户带来的新能力和价值

### 2.1 具体使用的飞书功能模块

#### 核心沟通与协作功能
- **飞书即时沟通**：统一内部沟通渠道，实现多种功能打通和整合
- **飞书日历**：提供"一键创建会议纪要"功能
- **飞书文档**：支持信息沉淀和知识管理
- **飞书OKR**：用于设定和管理团队目标

#### 专业化服务功能
- **飞书服务台**：创建IT服务台，通过智能答疑和人工答疑结合处理内部疑问
- **飞书知识库**：用于信息管理和知识沉淀
- **飞书绩效**：支持绩效管理工作
- **飞书工作台/开放平台**：支持自研系统接入，实现报警平台与沟通工具的无缝连接

### 2.2 解决的主要业务痛点

#### 痛点一：横向沟通效率低下
**问题描述**：
- 与上下级关系明确的纵向沟通不同，横向沟通指各部门内员工及跨部门之间的沟通和配合
- 对于组织结构扁平化的互联网公司来说，横向沟通是更普遍的沟通场景
- 此前使用过的工具都未能为此提供足够支持
- 存在信息割裂、协作低效等问题

**解决方案**：
- 借助飞书的即时沟通功能，统一内部沟通渠道
- 实现与日历、文档等多种功能的打通和整合
- 利用OKR应用设定和管理团队目标，各部门成员间可以清晰了解彼此的工作重点
- 方便横向对齐目标与进度

#### 痛点二：信息沉淀和管理困难
**问题描述**：
- 团队每天都有大量繁杂的信息交流
- 对这些信息缺乏及时的沉淀和管理
- 很多想法在提出后都"石沉大海"难以追溯
- 亟需借助工具来帮助提炼与整合

**解决方案**：
- 利用飞书日历中的"一键创建会议纪要"功能
- 及时记录沟通过程中的关键信息和待办事项
- 更高效地推动具体任务的执行和落地
- 记录无需在文件中翻找，只要打开相关日程或会议群组即可快速查看

#### 痛点三：IT支持效率低下
**问题描述**：
- 各部门成员都或多或少会遇到一些IT方面的问题
- 尤其是有新人入职时，答疑需求比较多
- 员工遇到IT问题时，会在全员群中@IT同学
- 信噪比低且不利于问题的追踪和落实

**解决方案**：
- 在飞书上创建IT服务台
- 通过智能答疑和人工答疑的结合，高效处理内部疑问
- 避免过多消息对无关成员的打扰
- 通过服务台清晰地了解内部成员对于IT的常见问题
- 更便捷地统计和评估IT团队的工作量

#### 痛点四：实时监控和响应机制不完善
**问题描述**：
- 同时运营多个产品，需要实时监控各个产品线的情况
- 需要在发生异常情况时及时通知相关负责人
- 自研了报警平台，但需和沟通工具无缝连接才能保证信息的传达和响应

**解决方案**：
- 在飞书工作台中接入自研的App监控报警系统
- 及时监测各个App的运营情况并将异常信息实时推送到飞书中
- 及时通知到相关负责人，提升团队的响应和处理速度

### 2.3 带来的业务价值和效率提升

#### 组织协作效率提升
- **横向沟通优化**：通过多功能打通和OKR对齐，显著提升了部门间协作效率
- **信息管理规范化**：建立了一套完整的信息沉淀和检索机制
- **响应速度提升**：通过报警系统与飞书的集成，大大提升了异常情况的响应速度

#### 运营管理优化
- **IT支持专业化**：从全员群混乱提问转向专业化服务台管理
- **工作量可量化**：能够更便捷地统计和评估IT团队的工作量
- **问题追踪能力提升**：建立完整的问题追踪和落实机制

#### 知识管理改进
- **会议纪要标准化**：通过一键创建功能，确保会议信息得到及时记录
- **信息检索便捷化**：相关信息可以通过日程或会议群组快速查看
- **经验积累系统化**：避免重要信息和想法"石沉大海"
