雅迪科技集团有限公司

### 所属行业
制造业 - 电动车/两轮车制造

## 2. 飞书给客户带来的新能力和价值

### 2.1 具体使用的飞书功能模块

1. **飞书多维表格** - 在三大核心业务场景中均有应用
   - 400客服工单数据处理
   - 行政服务管理(礼品领用、通勤车借用)
   - 生产异常处理直通车

2. **飞书服务台** - 主要用于行政服务场景
   - 员工行政咨询统一入口
   - 机器人自动解答重复问题

3. **飞书多维表格AI功能** - 用于行政服务创新
   - 车牌自动识别功能(寻人挪车场景)

### 2.2 解决的业务痛点

#### 2.2.1 售后客服业务痛点
- **数据处理繁琐且汇报直观性欠佳**：此前依赖Excel处理数据，需手动输入并编写大量公式计算，耗时易出错
- **报表制作任务繁重且时效性不足**：需安排2名专人负责报表制作，报表完成后时效性差
- **数据难以有效支撑客服考核工作**：从海量工单数据中统计、排名客服表现困难，考核缺乏数据支撑

#### 2.2.2 行政服务业务痛点
- **咨询答疑难题**：员工行政咨询缺乏统一渠道，行政人员需重复解答相同问题
- **寻人挪车繁琐**：员工车辆被挡时，需多渠道找车主，耗时费力且有隐私泄露风险
- **信息收集管理落后**：行政业务登记多依赖手工台账，易重复、遗漏，纸质登记存在信息泄露风险

#### 2.2.3 生产流程业务痛点
- **设备与系统短板**：老厂产线数字化程度低，设备陈旧老化，型号繁多且接口协议不一致，现有mass系统处理异常功能不足
- **异常处理流程繁冗**：生产异常处理流程复杂，信息传递易出错，涉及多岗位和多部门协调，流程冗长繁琐

### 2.3 带来的业务价值和效率提升

#### 2.3.1 售后客服场景
- **数据自动化处理与同步更新**：客服系统数据无缝同步，预置关键指标逻辑公式自动计算
- **报表制作时效性提升**：可视化仪表盘实时自动刷新，释放2名专人人力
- **客服考核科学化**：仪表盘直观展示接单量、客户满意度、接单效率等多维度数据，快速定位异常工单
- **售后效率提升超过80%**

#### 2.3.2 行政服务场景
- **咨询服务数字化**：机器人自动解决约50%重复咨询问题，降低人力成本约12.5%
- **AI寻人挪车创新**：车牌照片自动识别车主信息并发送提醒，集团总部每周节省600分钟寻人挪车时间
- **信息收集管理数字化**：礼品领用每月节省约3.7人天工作量，零成本替代纸质登记单
- **量化收益**：平均每月节省人力成本16,875元，预计提升总体行政服务满意度5%

#### 2.3.3 生产流程场景
- **信息规范高效传递**：建立统一提报标准，确保生产异常信息格式与内容规范一致
- **任务协同透明化**：清晰设定各任务处理进度节点，提高协同效率
- **责任落实精准长效**：每项异常处理任务责任明确到具体岗位人员，形成有效闭环管理
- **经济效益显著**：月残车数从200多台降至3台后清零，单个基地提效171万元，预计复制到八大基地提效近千万

## 3. 飞书未能满足的客户需求

### 3.1 从案例中明确提到的需求

1. **生产设备数据采集和传输的局限性**
   - 案例中提到"设备采购分散、标准不一，兼容性差，导致新自动化设备难以接入生产线，数据采集与传输困难"
   - 这表明飞书在直接对接工业设备、自动化数据抽取和实时报警方面存在能力局限
   - 需要依赖其他系统或接口来完成底层设备数据采集

2. **现有系统集成复杂性**
   - 雅迪仍使用现有的mass系统处理部分生产管理功能，但该系统"功能单一，异常处理时无法精准定位问题、分配任务"
   - 飞书主要是在现有系统基础上构建异常处理流程，而非完全替代原有系统
   - 这表明在某些专业领域，飞书可能需要与专业系统协同工作

3. **复杂生产流程的深度集成需求**
   - 生产异常处理涉及多个部门、多个岗位的复杂协调
   - 飞书虽然优化了信息传递和任务分配，但底层的生产流程控制和质量监控可能还需要更专业的制造执行系统(MES)支持

### 3.2 从案例中隐含的未满足需求

1. **更深入的AI应用场景**
   - 虽然已应用AI进行车牌识别，但在生产质量预测、设备故障预警、客服意图识别等更复杂的AI场景似乎还有提升空间
   - 可以探索更多AI驱动的预测性分析和智能决策支持

2. **更全面的数字化工厂解决方案**
   - 飞书主要解决了信息流和协作流的问题，但在生产过程控制、质量追溯、供应链协同等制造业核心环节可能还需要更专业的解决方案
   - 完整的数字化工厂需要与ERP、MES、SCM等系统深度集成

3. **移动端和现场作业支持**
   - 案例中没有明确提到车间现场人员的移动应用场景，在制造业现场作业、设备巡检、质量控制等场景可能还需要更多移动端支持

4. **数据安全和合规性**
   - 作为制造业企业，在数据安全、知识产权保护、行业合规等方面可能有更高要求，案例中未提及飞书在这些方面的能力

### 3.3 可能需要的其他解决方案

1. **专业制造执行系统(MES)**
   - 用于生产计划、过程控制、质量管理和设备管理
   - 与飞书形成互补，解决生产过程中的专业管理需求

2. **企业资源规划(ERP)系统深度集成**
   - 用于物料管理、成本控制、财务核算等企业级管理
   - 与飞书的数据打通可以实现更全面的企业管理

3. **物联网(IoT)平台**
   - 用于设备数据采集、实时监控和预测性维护
   - 与飞书集成可以实现设备异常的及时通知和处理

4. **专业质量管理系统(QMS)**
   - 用于质量策划、质量控制和质量改进
   - 与飞书协作可以提升质量问题的处理效率和闭环管理

## 4. 战略洞察与建议

### 4.1 飞书在制造业的价值定位

从雅迪案例可以看出，飞书在制造业的价值主要体现在：

1. **信息协作平台**：解决制造业中信息传递不畅、协作效率低的问题
2. **流程数字化工具**：将传统线下流程数字化，提高效率和透明度
3. **数据可视化助手**：通过多维表格和仪表盘实现业务数据的直观展示
4. **AI轻应用场景**：在特定场景(如车牌识别)提供便捷的AI解决方案

### 4.2 对飞书的建议

1. **加强制造业专业场景解决方案**
   - 开发更多针对制造业特定场景的模板和最佳实践
   - 提供更专业的制造行业咨询和实施服务

2. **深化AI在制造业的应用**
   - 探索质量预测、设备故障预警等更复杂的AI应用场景
   - 提供更灵活的自定义AI功能，让企业可以根据自身需求定制

3. **增强系统集成能力**
   - 提供更开放的平台和API，便于与制造业专业系统(如MES、ERP)集成
   - 开发更多预制连接器，降低集成复杂度

4. **完善移动端现场作业支持**
   - 开发更适合车间环境的移动应用界面
   - 提供离线操作和数据同步能力，适应复杂的工厂网络环境

### 4.3 对制造业客户的建议

1. **分阶段推进数字化转型**
   - 从协作和信息流数字化入手，逐步推进到生产过程数字化
   - 优先选择高价值、易实现的场景进行试点

2. **重视数据标准化和流程规范化**
   - 在数字化过程中建立统一的数据标准和流程规范
   - 确保数字化的基础质量，为后续深化应用奠定基础

3. **构建生态系统思维**
   - 将飞书作为数字化平台的一部分，与其他专业系统协同工作
   - 重视不同系统间的数据流通和业务协同

## 5. 总结

雅迪案例展示了飞书在制造业数字化转型中的典型应用场景和价值实现。通过在售后客服、行政服务和生产异常处理三个关键业务环节的实践，雅迪实现了显著的效率提升和成本节约，验证了飞书作为企业协作和流程数字化平台的价值。

同时，案例也反映出飞书在制造业应用中的一些局限性，特别是在专业制造系统集成、复杂生产流程控制等方面还需要与其他专业系统协同。未来，随着AI技术的发展和制造业数字化需求的深化，飞书可以在更多制造业场景中发挥更大价值，但也需要持续增强专业能力和行业解决方案。
