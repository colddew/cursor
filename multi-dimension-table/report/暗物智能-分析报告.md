## 飞书给客户的新能力/赋能

### 1. 兼容二元项目管理模式
- **研发型项目管理**：按照「Epic-Spring-Story」层层拆解，保持敏捷管理模式，清晰展示项目进展
- **交付型项目管理**：以「项目管理」为主线的瀑布式管理，通过里程碑计划、风险变更管理保障项目有序推进
- **统一平台管理**：一个工具同时兼容「敏捷式」与「瀑布式」的管理模式，解决以往需要分开管理的问题

### 2. 流程自动化与信息提效
- **自动任务推送**：通过飞书项目的自动化能力，为每个项目成员自动推送任务，任务完成情况一目了然
- **流程驱动过程**：任务完成后流程自动流转节点，既用流程驱动过程，又让过程反哺流程
- **信息自动聚合**：通过工具实现信息自动聚合、分发与多维度展示，减少对单点"人"的依赖

### 3. 资源管理与知识支持
- **资源排期在线管理**：研发人员可通过任务维度进行排期和工作量评估，管理方可通过「任务」、「人员」双维视角进行资源管理
- **工时登记系统**：结合飞书项目开放平台，自定义工时管理插件，实现在线化工时管理
- **PMO小助手服务台**：搭建「自助查询、人工咨询、知识推送」一体化的服务窗口，有效提高问题处理效率

### 4. 全流程协同管理
- **售前-研发-交付串联**：将售前项目、产品研发、交付落地三大环节串联，保障信息全在线、资料不丢失、目标一致、协同高效
- **规范化商机管理**：通过不同阶段的规范化录入，为管理决策提供依据，为商务跟进提供指引
- **自动流程流转**：商务在线填写售前项目信息后，系统自动生成后续流转环节，实现流程自动化

### 5. 具体成效数据
- **协同效率提升20%**：通过流程驱动过程、信息流动提效实现
- **人工提问率减少50%**：在「PMO小助手」帮助下，有效提高问题处理效率
- **交付过程100%留痕**：保障三大环节串联，确保信息完整记录
- **支持上百个在途项目**：实现"一个人管理上百个在途项目"的系统化运作

## 飞书没有满足客户需求的部分

### 1. 工具定制化需求
- **工时管理插件需自主开发**：需要结合飞书项目的开放平台，根据自身管理需求自定义工时管理插件
- **特定行业适配**：作为AI技术公司，可能需要更多针对研发项目和AI项目的特定功能模块

### 2. 系统集成复杂度
- **考勤系统集成**：工时管理需要结合考勤信息，表明飞书原生功能与客户现有系统需要进一步整合
- **项目管理模板库维护**：需要为不同项目模式建立标准模板库，维护工作仍需人工投入

### 3. 高级功能扩展
- **复杂流程定制**：对于售前到交付的复杂流程，需要系统根据信息自动生成后续流转环节，可能超出飞书标准功能范围
- **多维度资源管理**：需要通过「任务」、「人员」的双维视角进行管理，可能需要额外开发或配置

### 4. 行业特定需求
- **AI研发项目特殊管理**：作为AI技术公司，可能需要更多针对算法研发、模型训练等特殊工作流程的管理功能
- **知识产权保护**：技术型企业在知识管理、权限控制等方面可能有更高要求
