# 得到APP飞书客户案例分析报告

## 1. 客户基本信息

**公司名称：** 思维造物（得到App）

**所属行业：** 文娱传媒 - 终身教育服务

**公司规模：** 
- 线下校区覆盖国内10个城市
- 开设112个班次
- 录取学员超过9,000人
- 现场参与跨年演讲累计超过4万人

**主营业务：**
思维造物是一家从事"终身教育"服务的企业，秉承"建设一所全球领先的终身学校"的愿景，以"让每个人都能从知识中获得力量"为使命。公司通过多种平台提供知识服务：

**线上业务：**
- 得到App：提供课程、听书及电子书等产品
- "罗辑思维"微信公众号

**线下业务：**
- "得到高研院"：为职场人士提供线上课程学习、线下实践转化和社交的创新学习方式
- "时间的朋友"跨年演讲：首创"知识跨年"新形态

## 2. 飞书给客户带来的新能力和价值

### 2.1 具体使用的飞书功能模块

1. **飞书文档**：核心协作工具，承载课程生产全流程
2. **飞书妙记**：音视频转文字工具，用于访谈记录和内容整理
3. **飞书知识库**：沉淀和检索知识内容
4. **飞书绩效**：支持绩效管理相关功能

### 2.2 解决的业务痛点

#### 痛点一：课程教研流程低效
**原有问题：**
- 课程文稿使用Word格式，通过微信在讲师和教研之间来回流转
- 教研需上传到石墨，通过企业微信发送总编室反复确认
- 一节课需要300次校对，涉及3个部门，在4个工具上流转
- 生成多个版本，流通范围难以控制

**解决方案：**
- 使用飞书文档实现"一篇文档"带动"一讲课程"的完整生产
- 讲师用飞书文档撰写课程大纲，同一文档在教研和总编室间流转
- 通过严格的权限管控和便捷的分享授权机制，在可控范围内高速流转

#### 痛点二：访谈内容整理耗时
**原有问题：**
- 教研需要与讲师进行高频访谈（每节课30讲，每讲至少3次，每次2小时）
- 需要反复回听访谈录音，手敲沉淀成文字稿件
- 基本是"人工速记"模式，效率低下

**解决方案：**
- 使用飞书妙记自动将音视频内容转录为可搜索、可翻译、可定位的智能文字笔记
- 教研访谈后点击录屏按钮，会后立刻获得访谈逐字稿
- 通过搜索关键词迅速定位关键内容，节省将近1倍的回听时间

#### 痛点三：传统组织协作模式限制
**原有问题：**
- 传统企业受权力、功能和层级的限制，难以激发员工创造力
- 员工价值感和参与度不足

**解决方案：**
- 实现以"事儿"为中心的协作逻辑
- 每个员工都可以扮演两个角色："事儿的建立者和领导者"和"事儿的帮助者和贡献者"
- 人围绕文档来转，通过文档推动"事儿"往前进

#### 痛点四：周报形式主义严重
**原有问题：**
- 传统周报流于形式，员工为"取悦上级"而写
- 管理者没有足够时间阅读和反馈大量周报
- 周报未能实现有效的信息传递和项目管理

**解决方案：**
- 创新采用"上级写给下级"的周报模式
- 脱离"取悦上级"属性，转为上级与下属主动沟通
- 实现信息在组织内透明与开放，促进知识自上而下流动

### 2.3 业务价值和效率提升

#### 量化价值：
1. **知识管理效率提升：**
   - 300篇：所有知识接入飞书知识库沉淀文档数
   - 50%↓：层级式的知识库页面树让知识检索时间减少
   - 10%+↓：全局搜索穿透知识库文档让新人培养成本降低

2. **教研流程优化：**
   - 节省将近1倍的访谈回听时间
   - 每节课可节省600人天的整理时间
   - 实现一节课程内容只用一篇文档承载，避免多版本混乱

#### 质化价值：
1. **组织文化升级：**
   - 从传统层级式组织转向以"事"为核心的协作组织
   - 激发员工自发、主动的工作态度
   - 提升员工价值感和满足感

2. **知识流动优化：**
   - 实现知识自上而下有效流通
   - 管理层通过周报分享心得、体会和视野情报
   - 建立透明开放的信息共享机制

3. **协作模式创新：**
   - 人围绕文档来转，而非文档围着人转
   - 建立"事儿"驱动的协作逻辑
   - 提升跨部门协作效率

### 2.4 特色应用场景

#### 周报模板创新：
得到设计了独特的周报模板，包含四条内容：
1. 上周工作
2. 下周计划  
3. 心得和体会（要求对工作有深刻思考）
4. 视野和情报（要求对行业和外部环境保持敏锐）

这种模板设计驱使管理者不断思考和学习，同时将高层信息有效传达给下属。

## 3. 飞书未能满足的客户需求

### 3.1 已识别的需求缺口

从案例内容来看，得到APP作为知识创作型企业，可能还存在以下未被完全满足的需求：

1. **内容分发与用户互动整合：**
   - 案例主要聚焦内部协作和内容生产，但未提及如何将内部创作内容高效分发到得到App平台
   - 缺乏内部协作平台与外部用户平台之间的无缝连接

2. **知识版权保护机制：**
   - 作为知识付费平台，内容的版权保护和安全分发至关重要
   - 案例未提及飞书在内容版权保护和安全管理方面的具体支持

3. **用户反馈与内容迭代闭环：**
   - 缺乏将用户学习反馈快速回流到内容创作流程的机制
   - 内部协作与外部用户数据之间可能存在断层

4. **多平台内容同步：**
   - 得到拥有多个内容平台（App、公众号、线下课程等）
   - 案例未提及如何通过飞书实现多平台内容的统一管理和同步

### 3.2 潜在的功能改进空间

1. **更深入的教育行业定制功能：**
   - 针对教育行业的特殊工作流程提供更定制化的解决方案
   - 课程开发、学员管理、教学效果评估等垂直场景的深度整合

2. **内容质量管控工具：**
   - 针对知识内容的审核、校对、质量评估提供更专业的工具
   - 建立标准化的内容生产质量控制流程

3. **知识图谱和关联推荐：**
   - 利用AI技术建立内部知识图谱，促进知识的关联和再利用
   - 基于历史内容智能推荐相关素材和参考资料

### 3.3 可能需要的其他解决方案

1. **CRM系统集成：**
   - 需要将内部协作与客户关系管理相结合
   - 学员信息管理与教学内容个性化的联动

2. **学习分析系统：**
   - 需要专业的学习数据分析工具
   - 将用户学习行为数据反馈到内容优化过程

3. **多媒体内容管理：**
   - 得到的内容形式多样（音频、视频、图文等）
   - 可能需要更专业的多媒体内容管理和制作工具

4. **移动端创作工具：**
   - 支持讲师和教研人员随时随地进行内容创作和协作
   - 移动端与PC端的无缝同步

## 4. 战略洞察

### 4.1 飞书的核心价值主张
得到案例充分展示了飞书在知识密集型企业中的核心价值：
- **内容生产效率**：通过文档协作和妙记工具大幅提升内容创作效率
- **组织协作升级**：打破传统层级限制，建立以"事"为中心的协作模式
- **知识管理优化**：通过知识库和搜索功能实现知识的高效沉淀和检索

### 4.2 知识教育行业的适配性
飞书特别适合知识教育行业的特点：
- **内容导向**：以文档为核心的生产流程与教育内容创作天然匹配
- **协作密集**：教育内容生产需要多角色、多环节的紧密协作
- **知识积累**：教育企业天然需要知识的沉淀、复用和传承

### 4.3 创新应用模式的启示
得到的周报模式创新给其他企业提供了重要启示：
- **管理创新**：工具的使用应该促进管理思维的创新，而不仅仅是流程的数字化
- **文化变革**：数字化工具的使用需要配套的组织文化变革
- **价值重构**：重新思考传统工作方式的价值和意义
