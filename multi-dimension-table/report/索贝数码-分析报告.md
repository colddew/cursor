## 2. 飞书给客户带来的新能力和价值

### 2.1 具体使用的飞书功能模块

1. **飞书多维表格**：核心应用模块，用于多个业务场景
2. **飞书审批**：用于业务申请和盖章审批流程
3. **飞书知识库**：用于产品文档集中管理
4. **飞书自动化工具**：用于自动任务创建和提醒
5. **飞书词典**：用于术语管理

### 2.2 解决的业务痛点

1. **数据孤岛问题**：原有多套OA和数字化工具各自独立，数据难以串联
2. **收款项目管理低效**：年均处理千个项目、上亿金额，但使用电子表格、即时通讯和邮件效率低
3. **审批流程繁琐**：业务申请场景复杂，线下审批耗时长、难溯源
4. **生产任务跟踪困难**：多平台管理、手工记录、周报反馈导致进度难实时掌握
5. **产品文档管理混乱**：安装配置文档权限获取难、内容难懂，导致交付周期长

### 2.3 带来的业务价值和效率提升

#### 2.3.1 收款管理优化
- **量化成果**：1张表管理每年上千个收款项目，累计金额达上亿元
- **具体改进**：
  - 数据隔离：各事业部数据清晰划分，保障信息安全
  - 自动化任务：收款计划自动下发至责任人，无需人工干预
  - 实时动态更新：回款进度实时呈现，数据自动汇总整合
  - 协同效率提升：多部门共享表格实时更新，解决数据分散问题
- **分析功能**：
  - 仪表盘监控：直观展示各事业部/分公司回款进度
  - AI深度分析：通过智能算法挖掘收款数据，生成精准报告，辅助科学决策

#### 2.3.2 审批效率提升
- **量化成果**：业务类型审批从原需1-2天缩短至30分钟左右完成，效率提升约1900%
- **具体改进**：
  - 多条件分支审批路径，满足同时发起多个盖章审批
  - 支持移动审批，耗时更短
  - 借助多维表格与飞书审批的共享数据源功能，完成审批数据溯源与归档完善
  - 实现多部门协同高效办公

#### 2.3.3 沟通成本降低
- **量化成果**：减少了跨部门沟通冗余，降低了50%的沟通成本
- **具体改进**：
  - 智能提醒机制：任务到期自动发送提醒
  - 定期汇报推送：按预设规则向管理层自动推送周/月度回款进度

#### 2.3.4 商务接待管理数字化
- **量化成果**：实现全流程数字化，减少了20%的能耗浪费
- **具体改进**：
  - 流程创建：在飞书审批中心搭建商务接待申请审批流程
  - 沟通协同：借助飞书自动化工具，将审批通过的接待需求自动转化为任务工单
  - 数据处理：运用多维表格的数据源获取功能，自动抓取审批通过的商务接待申请数据
- **业务价值**：
  - 数据驱动决策：多维表格的可视化看板帮助管理层实时监控行政数据
  - 流程标准化：所有行政流程均实现了标准化和数字化
  - 跨部门协同：通过飞书平台的多部门协同功能，提升了跨部门协作效率

#### 2.3.5 生产任务管理优化
- **时间成本改进**：基于上述三类场景，自周报变为日报，且无需人为汇总及汇报，颗粒度更加细密
- **人力成本节省**：
  - 减少了周报维护、发送的工作量投入
  - 减少了进度更新口头提醒的沟通成本
- **效率提升**：
  - 干系人可随时依据项目或任务风险点，更早、更及时的介入或管控项目风险
  - 部门经理可根据仪表盘排行榜，更科学的平衡资源调度
  - 产品经理可根据筛选，随时统计出所属产品的销售数量和对应项目

#### 2.3.6 产品文档管理提升
- **直接效益**：
  - 时间成本：知识库上线后，一线交付在自主实施安装的周期极大缩短
  - 人力成本：一线技术人员在差旅成本上会随着交付周期缩短而节约
  - 效率提升：一线人员能自主安装后，将降低对总部生产部门的生产依赖
  - 文档管理：文档维护的便利性、统一性得到极大提高
- **业务价值**：
  - 为一线交付技术体系，打开了自主独立安装，从获取到集成的全文档路径支撑
  - 降低了总部生产压力和产品售后支持压力
- **复用价值**：单个产品知识库上线后，其业务模式可照搬其他数十个产品线

   - 需求：飞书需要与客户现有系统进行更深度的整合，打破系统间壁垒

2. **行业特定功能缺失**
   - 作为专业视听产品和数字化解决方案提供商，索贝有特定的行业需求：
