# 汉得信息飞书客户案例分析报告

## 客户基本信息

### 公司名称
上海汉得信息技术股份有限公司（简称"汉得"）

### 所属行业
企业数字化综合服务/IT咨询服务

### 公司规模
- 成立时间：1996年（前身为上海汉得计算机服务有限公司）
- 上市情况：2011年在深圳证券交易所创业板上市（证券代码：300170）
- 人员规模：未明确提及具体数字，但从案例描述可判断为大型企业，拥有专业的IT咨询顾问团队

### 主营业务
- 企业数字化综合服务商
- 致力于成为连接企业管理与数字化技术的桥梁
- 帮助企业实现数字化转型，提高运营效率、效益与竞争能力
- 与国际顶级IT厂商稳健合作，为客户提供全球领先的企业信息化建设实践经验

## 飞书给客户带来的新能力和价值

### 具体使用的飞书功能模块

1. **飞书多维表格**
   - 项目管理工作台搭建
   - 可视化任务管理
   - 轻量化CRM系统搭建
   - 资源申请和调配系统
   - 项目状态监控看板

2. **机器人助手**
   - 自动化流程触发
   - 节点自动检查
   - 项目调整自动通知
   - 智能问答服务台

3. **飞书知识库**
   - 项目交付知识空间
   - 各个项目阶段和类型解决方案存储
   - 权限分级管理
   - 知识资产保护

4. **飞书视频会议**
   - 国际业务交流
   - 实时翻译功能
   - 自动生成会议纪要

5. **飞书服务台**
   - 智能问答服务
   - 员工日常问题解决方案推荐

6. **飞书文档**
   - 文档翻译功能
   - 跨语言协作
   - 知识管理和链接

7. **飞书会议室**
   - 支持远程交付
   - 降低进场时间

### 解决的业务痛点

1. **项目管理痛点**
   - 项目过程不够透明
   - 进展风险依赖个人经验
   - 管理者难以全面掌握项目信息
   - 实施模板不统一
   - 资源匹配困难
   - 人员切换成本高
   - 风险管理依赖人工预警

2. **集中交付挑战**
   - 流程不统一
   - 需与现场高度协同
   - 无法有效记录开发迭代中的任务流转和责任归属
   - 敏捷设计和SAP实施开发思路差异
   - "跟催为王"的低效情况

3. **跨国协同障碍**
   - 语言障碍限制参会人员参与度
   - 信息传递受语言因素影响
   - 会议讨论和决策记录困难
   - 海外当地难以短期培养大量资深顾问

4. **创新推广困难**
   - 自上而下收集需求难以全面考虑一线顾问需要
   - 系统使用频率低
   - 数字化最后一公里难以打通

### 业务价值和效率提升（数据支撑）

1. **显著的数据提升**
   - 20%：项目资源利用率提高
   - 50%：项目资源调配效率提升
   - 25%：跨国业务沟通效率提升
   - 70%：系统搭建和运维成本降低
   - 20%：项目沟通时间精力节省

2. **管理效率提升**
   - 从"人找事"到"事主动找人"的转变
   - 项目信息实时同步，监控不再滞后
   - 项目经验直接复制、落地，翻倍项目启动速度
   - 可视化、多维度的任务管理提高项目透明度

3. **资源配置优化**
   - 系统自动推荐合适资源，减少线下沟通时间
   - 资源占用锁定功能避免资源冲突和浪费
   - 提升项目协同效率，保障项目任务优先级合理分配

4. **知识管理和创新**
   - 资深顾问经验固化在工具中，赋能更多员工
   - 支持一线顾问作为"开发者"搭建轻系统
   - 涌现大量"效率先锋"，自主搭建数十个轻量级系统

## 飞书未能满足的客户需求

### 明确提到的限制和挑战

1. **前期平台限制**
   - 案例中提到"前期汉得使用的平台无法有效记录开发迭代中的任务流转和责任归属"
   - 这表明在飞书之前使用的其他平台存在功能不足，但未明确说明飞书是否完全解决了这一问题

2. **不同项目类型管理需求的复杂性**
   - "不同项目类型的管理需求不同，难以实现不同项目之间的协同混编管理"
   - 虽然飞书提供了改善方案，但这种复杂性仍然是持续的挑战

3. **集中交付的敏捷设计和SAP实施开发之间的思路差异**
   - 飞书通过工具将方法论模板化帮助统一工作思路，但根本性的思路差异仍然存在

### 推断的潜在需求

1. **更深度的行业专业化需求**
   - 作为企业数字化综合服务商，汉得可能需要更多针对特定行业（如制造业、金融业等）的专业化功能模块
   - 飞书提供的通用解决方案可能无法完全满足特定行业的深度需求

2. **更复杂的项目管理需求**
   - 汉得处理的是企业级数字化转型项目，可能需要更复杂的项目管理功能
   - 如更精细的成本控制、更复杂的多级项目管理、更深入的风险评估等

3. **更强的定制化需求**
   - 案例显示汉得已经在自行搭建轻量级系统，表明可能有更强的定制化需求
   - 飞书的多维表格等工具虽然灵活，但对于极其复杂的业务场景可能仍有限制

4. **更深入的数据分析需求**
   - 作为数字化服务商，汉得可能需要更深入的数据分析和业务洞察能力
   - 飞书提供了AI分析功能，但可能无法满足极其复杂的业务分析需求

5. **更高级的安全和合规需求**
   - 作为处理客户敏感数据的IT服务商，汉得可能有更高级别的安全和合规要求
   - 虽然飞书提到了权限管理和知识资产保护，但可能还有更深层的安全需求

### 可能需要的其他解决方案

1. **专业的项目管理软件补充**
   - 对于极其复杂的企业级项目，可能需要专业的项目管理软件作为飞书的补充
   - 如更高级的甘特图、资源管理、成本控制等功能

2. **行业垂直解决方案**
   - 针对特定行业的深度需求，可能需要行业垂直的解决方案
   - 如制造业的MES系统集成、金融业的合规管理等

3. **更高级的数据分析平台**
   - 对于复杂的业务数据分析，可能需要更专业的数据分析平台
   - 如高级BI工具、数据挖掘平台等

4. **定制开发能力**
   - 对于极其特殊的业务需求，可能需要定制开发能力
   - 飞书的零代码平台可能无法满足所有复杂需求

## 战略洞察

1. **汉得信息作为数字化服务商的示范效应**
   - 汉得本身作为企业数字化服务提供商，选择飞书作为内部协作平台具有重要的示范意义
   - 这表明飞书的能力得到了专业数字化服务提供商的认可

2. **飞书在企业服务行业的定位**
   - 汉得案例展示了飞书在企业服务行业的强大适用性
   - 特别是在项目管理、跨国协作、知识管理等方面的价值

3. **微创新和自下而上创新的重要性**
   - 汉得案例突出了飞书支持"自下而上的创新"的重要性
   - 一线员工能够通过飞书工具自行搭建解决方案，这是传统IT系统难以实现的价值

4. **跨国企业的数字化转型需求**
   - 汉得的国际化业务展示了飞书在支持跨国企业数字化转型方面的能力
   - 特别是语言翻译、跨时区协作等功能的价值

5. **飞书与传统IT系统的关系**
   - 汉得案例表明飞书更多是作为现有自研系统的"重要补充和延伸"
   - 飞书不是完全替代传统IT系统，而是提供更灵活、更易用的补充功能
