## 📚 完整的技术演进记录

### 阶段1：本地方案尝试（失败）
- PaddleOCR 本地部署 → 安装困难、速度慢
- Tesseract → 安装慢、准确率低
- **结论**：本地方案不可行

### 阶段2：Apple Vision Framework
- 测试成功，速度快（2.6秒）
- **问题**：表格结构丢失
- **结论**：只适合文字提取

### 阶段3：百度 API 探索
- 百度通用 OCR → 表格能力一般
- 百度表格识别 V1 → 已废弃
- 百度表格识别 V2 → 免费额度少
- 百度文档解析（旧版）→ 被替代

### 阶段4：AI Studio PaddleOCR-VL（最终方案）
- 发现用户有 AI Studio 资源
- PaddleOCR-VL 表格识别 SOTA
- AI Studio：学习平台，无明确免费额度（超额会429）
- 百度智能云：商业平台，200页一次性免费
- **结论**：使用 AI Studio（用户已有资源）

### 阶段5：需求明确和架构优化
- 明确需求：500页招生大本
- 设计批量处理工具
- 设计合并工具
- 并发优化方案

## 📊 所有讨论的关键决策

1. **为什么不用本地 OCR？** → 安装困难、速度慢、准确率低
2. **为什么不用 Vision OCR？** → 表格结构丢失
3. **为什么选 AI Studio 而不是百度智能云？** → 免费资源、效果相同
4. **为什么选 PaddleOCR-VL？** → 表格识别最强（SOTA）
5. **为什么需要并发？** → 500页单线程需要2小时，并发可缩短到12分钟
6. **为什么需要人工分类？** → 不同省份格式差异大，自动识别不可靠
7. **为什么用图片而不是 PDF？** → 单页处理、可并发、可断点续传
